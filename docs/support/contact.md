## Support Channels

### Email Support

**support@oncallshift.com**

Best for:
- Account and billing questions
- Feature requests
- Non-urgent technical issues
- General inquiries

**Response times**:
- Starter: 24-48 hours
- Professional: 12-24 hours
- Enterprise: 4-8 hours

**Tips for faster resolution**:
- Include your organization name
- Describe the issue clearly
- Include steps to reproduce
- Attach screenshots if helpful
- Include relevant error messages

### Urgent Issues

For critical issues affecting your ability to respond to incidents:

**Email**: support@oncallshift.com with "URGENT" in subject line

**Include**:
- Organization name
- Description of impact
- What you've already tried
- Contact phone number

Enterprise customers have access to dedicated support channels.

## Report a Bug

Found a bug? Help us fix it:

1. **Check known issues**: [status.oncallshift.com](https://status.oncallshift.com)
2. **Search existing reports**: [GitHub Issues](https://github.com/oncallshift/oncallshift/issues)
3. **Submit new bug**:
   - Email: bugs@oncallshift.com
   - Or create a GitHub issue

**Include**:
- Steps to reproduce
- Expected vs. actual behavior
- Browser/device information
- Screenshots or screen recordings
- Console errors (if applicable)

## Security Concerns

### Report a Vulnerability

If you discover a security vulnerability:

**Email**: security@oncallshift.com

**Include**:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Your contact information (for follow-up)

**Our commitment**:
- Acknowledge within 24 hours
- Investigate promptly
- Keep you informed of progress
- Credit researchers (if desired)

Do not publicly disclose vulnerabilities until we've had time to address them.

### Security Documentation

- [Security Overview](https://oncallshift.com/security)
- [Privacy Policy](https://oncallshift.com/privacy)
